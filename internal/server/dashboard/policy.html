policy.yaml — canonical-generated; every save fully rewrites the file (comments and field-order will be rewritten)
YAML examples — copy, paste, edit

policy.yaml is an overlay on the built-in rules. With nothing set, the file is just version: 1 — that's correct, not a bug. Every section below is optional. Paste only what you need.

Rule overrides (severity bump, disable a rule, narrow scope):

version: 1
rules:
  # Disable a rule globally
  mcp-unpinned-npx:
    enabled: false
    notes: "Our team pins MCP servers via a different mechanism."

  # Re-rank a rule's default severity
  claude-third-party-plugin-enabled:
    severity: low
    notes: "Every plugin in our marketplace was security-reviewed."

  # Narrow a rule's scope to specific paths
  claude-hook-shell-rce:
    scope:
      include: ["~/.claude/**", "~/work/**"]
      exclude: ["~/work/sandbox/**"]

Named allowlists (string-sets rules can consult):

allowlists:
  trusted-mcp-servers:
    entries:
      - "@modelcontextprotocol/server-fs"
      - "@anthropic/mcp-server-github"
    notes: "Vendors we've audited and trust."

  vendor-plugin-paths:
    entries:
      - "~/.claude/plugins/cache/vercel/**"
      - "~/.cursor/extensions/**"
    notes: "Vendor caches — demote, don't suppress."

Suppressions (silence specific (rule, path) pairs):

suppressions:
  - rule: mcp-unpinned-npx
    path: "~/.cursor/**"
    reason: "Cursor manages its own MCP servers; their bundled config is trusted."

  - rule: dependency-osv-vulnerability
    path: "**/testdata/**/package-lock.json"
    reason: "Test fixtures pin vulnerable versions on purpose."

  - rule: secret-betterleaks-valid
    path: "~/projects/legacy-app/.env.example"
    reason: "Tracked sample env file with placeholder keys."
    expires: "2026-12-31T00:00:00Z"

Save runs through the same validate path the Form view uses. Errors surface inline; nothing lands on disk until validation passes.
